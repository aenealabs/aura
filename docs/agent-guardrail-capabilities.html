1. Guardrail Pipeline Overview

Guardrails are not a single layer — they form a defense-in-depth pipeline that wraps every agent interaction from input to output to execution. Probabilistic layers (input/critique) combine with deterministic layers (constraint coherence) for comprehensive coverage.

2. Semantic Guardrails Engine (ADR-065)

Six progressively deeper detection layers process every input with strict latency budgets. The engine detects 8 threat categories and produces one of 4 actions. Performance targets: P50 <150ms, P95 <300ms, P99 <500ms. 793 tests.

Layer 1: Canonical Normalization

Unicode NFKC normalization, homograph detection, zero-width character removal, multi-encoding decode (base64, URL, HTML entities), control character sanitization. Max 3 decode iterations to prevent decode bombs.

normalizer.py · Target: 5ms

Layer 2: Fast-Path Pattern Matching

Regex-based threat patterns and SHA-256 blocklist hash verification. Categories: system override, jailbreak, hidden instructions, delimiter injection, exfiltration. Fast-exit on CRITICAL or blocklist hit.

pattern_matcher.py · Target: 10ms

Layer 3: Embedding Similarity Detection

OpenSearch k-NN search (10 neighbors, min score 0.5) against threat corpus index. High-confidence threshold: 0.85 similarity. Query caching with 5-minute TTL.

embedding_detector.py · Target: 50ms

Layer 4: LLM Intent Classification

Multi-model classification (Haiku fast / Sonnet balanced / Opus accurate). Classes: Legitimate, Override, Exfiltration, Role confusion, False positive. Semantic cache integration (95% similarity threshold).

intent_classifier.py · Target: 150ms

Layer 5: Multi-Turn Session Tracking

DynamoDB-backed session state with exponential decay scoring (0.9 factor per turn). HITL escalation at cumulative score 2.5. Max 50 turns tracked per session. 24-hour TTL.

multi_turn_tracker.py · Target: 20ms

Layer 6: Decision Engine

Aggregates all layer results into a final action: ALLOW, SANITIZE, BLOCK, or ESCALATE_HITL. Block on any CRITICAL threat or multiple HIGH threats. Full audit logging to DynamoDB queue.

decision_engine.py · Target: 5ms

4. Constraint Geometry Engine (ADR-081)

Deterministic cortical discrimination across a 7-axis constraint space. Unlike the probabilistic semantic guardrails, the CGE produces the same coherence score for the same input every time. 4 policy profiles map scores to actions. 358 tests.

AxisConstraintWhat It Checks
C1Syntactic ValidityCode structure, AST validation, parsing correctness
C2Semantic CorrectnessLogic verification, type consistency, intent alignment
C3Security Policy (NIST 800-53)Control compliance, threat patterns, vulnerability checks
C4Operational BoundsResource limits, blast radius constraints, deployment rules
C5Domain ComplianceIndustry-specific requirements, regulatory alignment
C6Provenance TrustSource integrity (ADR-067), context authenticity, trust adjustment
C7Temporal ValidityTime-bound constraints, expiration rules, schedule compliance

Coherence Scoring

Cosine similarity against positive/negative centroids. Harmonic mean for rule aggregation, geometric mean for axis aggregation. Normalized to [0.0, 1.0].

4 Policy Profiles

ProfileWhen
AUTO_EXECUTECCS above threshold — auto-approve
HUMAN_REVIEWCCS in range — requires review
ESCALATECCS low — human + audit trail
REJECTCCS critical failure — block
