2. Five-Tier Cognitive Memory

Inspired by human cognitive neuroscience, Aura implements five distinct memory types that work together to give agents persistent learning, contextual awareness, and adaptive behavior.

💭
Working Memory

Active task context held during execution. Capacity-limited to Miller's Law (7±2 items). Manages attention, token budgets, and intermediate results. Evicts lowest-salience items when full. Cleared on task completion.

📖
Episodic Memory

Full context snapshots of past problem-solving instances. Records task description, input context, decision, reasoning, confidence, and outcome (SUCCESS/FAILURE/PARTIAL/ESCALATED). 30-day default TTL with importance-weighted retention.

📚
Semantic Memory

Generalized knowledge extracted from episodes: guardrails, patterns, schemas, concepts, and anti-patterns. Severity-ranked (CRITICAL/HIGH/MEDIUM/LOW). Auto-extracted when 3+ episodes show consistent patterns with 85% validation accuracy.

⚙️
Procedural Memory

Action sequences and workflows with per-step success tracking. Records conditions, actions, and validation logic. Domain-scoped with success rate metrics. Enables agents to replicate successful remediation patterns.

🔮
Neural Memory (Titan)

Deep MLP with test-time training (TTT) and MIRAS configuration. Provides sub-symbolic pattern generalization that complements the symbolic memory types. Surprise-driven selective memorization based on gradient magnitude.

3. Titan Neural Memory (ADR-024)

The Titan Neural Memory implements a deep MLP memory model with test-time training (TTT), enabling agents to learn at inference time via gradient descent. Surprise-driven selective memorization ensures only novel, important information is retained.

Three-Tier Compute Strategy

Retrieval Path

AWS Inferentia2 (inf2) instances for optimized, low-latency inference operations.

TTT Path

GPU (g5) instances for gradient computation during test-time training updates.

Development

CPU fallback (t3) instances for cost-effective iteration and testing.

7. ReMem Action Framework

The 7 ReMem operations provide a comprehensive toolkit for memory evolution, from basic maintenance (consolidate, prune) to advanced learning (abstract, link) and safety (correct, rollback).

CONSOLIDATE Phase 1a

Merges similar experiences using cosine, Jaccard, Levenshtein, and semantic similarity metrics. Clustering parameters configurable per domain.

Low latency | Reduces memory redundancy

PRUNE Phase 1a

Removes low-value memories based on access frequency, age, and relevance score. Retention policies: recency-weighted or importance-based.

Low latency | Prevents memory bloat

REINFORCE Phase 1b

Strengthens successful patterns via Titan test-time training. Updates persistent memory weights and tracks success rates and confidence.

Titan TTT integration | Adaptive learning

ABSTRACT Phase 3

Uses LLM to extract generalizable strategies from specific episodes. Creates domain-specific schemas and patterns from experience.

LLM-powered | Knowledge generalization

LINK Phase 5

Creates cross-memory associations as Neptune graph edges. 7 link types: STRATEGY_DERIVED_FROM, REINFORCES, CONTRADICTS, SUPERSEDES, RELATED_TO, DEPENDS_ON, SIMILAR_TO. Confidence-scored 0.0–1.0.

Neptune graph | Associative memory

CORRECT Phase 5

Fixes incorrect memories with LLM verification. 6 correction reasons: factual error, outdated info, misattribution, strategy failure, user feedback, LLM verification. Human review flag for uncertain corrections.

LLM-verified | Error correction

ROLLBACK Phase 5

Restores memories from DynamoDB Streams snapshots. 4 snapshot sources: DYNAMODB_STREAMS, POINT_IN_TIME (PITR), MANUAL_BACKUP, PRE_OPERATION. Full point-in-time recovery support.

DynamoDB Streams | Safety net

8. Multi-Agent Memory Sharing

The Multi-Agent Memory Sharing Service (1,000 LOC) enables agents to share learned patterns, propagate successful strategies, and pool experiences while maintaining tenant and security domain boundaries.

Cross-Agent Discovery

Agents discover and subscribe to relevant memory streams from other agents based on capability overlap and domain relevance.

Pattern Propagation

When one agent learns a successful pattern (via ABSTRACT or REINFORCE), it can be propagated to agents with similar capabilities.

Experience Pooling

Multiple agents' episodes are pooled for more robust pattern extraction, reducing the number of episodes needed for generalization.

Conflict Resolution

When agents develop contradictory patterns, the sharing service detects and resolves conflicts using confidence scoring and success rates.

Security Boundaries: Memory sharing respects tenant isolation (tenant_id) and security domain boundaries (security_domain). Cross-tenant sharing is never permitted. Cross-domain sharing requires explicit policy configuration.

Subsystems

Pattern Completion Retriever

Retrieves memories based on partial or incomplete context using cosine similarity, weighted by recency. Merges candidates from all memory types, ranked by relevance.

Automatically converts episodes into semantic patterns: (1) Cluster similar episodes, (2) Find common keywords/decisions, (3) Validate with test episodes at 85% accuracy threshold, (4) Create semantic memory.

20. Agent Memory Lifecycle

Every agent follows a structured memory lifecycle during task execution, loading relevant memories before work begins and persisting learnings after completion.

1. INIT — Agent initialized with task parameters ↓ 2. MEMORY_LOAD — Load from all memory tiers: • Episodic: Recent similar task outcomes • Semantic: Relevant guardrails, patterns, schemas • Procedural: Known workflows for this task type • Neural (Titan): Sub-symbolic pattern retrieval • Working: Initialize with 7±2 capacity ↓ 3. CONTEXT_RETRIEVAL — Hybrid GraphRAG via Context Engineering Three-Way Retrieval → HopRAG → Context Scoring → Stack Assembly ↓ 4. EXECUTION — Agent performs task (CODE_GENERATION, SECURITY_REVIEW, etc.) Working memory updated with intermediate results Confidence estimated continuously from memory factors ↓ 5. MEMORY_STORE — Persist learnings: • Store episode to DynamoDB (full context snapshot) • Update Titan Neural Memory (test-time training if novel) • Trigger Evo-Memory evolution (consolidate, abstract, link) • Extract patterns if 3+ similar episodes exist ↓ 6. CONSOLIDATION — Post-task maintenance: • Memory pressure check and consolidation if needed • Metrics published to CloudWatch • Audit trail written for compliance

Checkpoint Architecture

The Agent Orchestrator (2,260 LOC) implements checkpoint/resume for long-running workflows. Each phase produces a serializable AgentCheckpoint containing workflow state, hybrid context data, and intermediate outputs, enabling recovery from failures at any phase.
