# Security Analysis: CKGE Core Ingestion Logic

**Target Component:** ASTParser and GraphBuilderAgent (Core Ingestion Logic)
**Reviewer Persona:** Senior Manager, Engineering Ops / Security Focus
**Goal:** Identify and plan remediation for vulnerabilities inherent in code processing logic, ensuring compliance with enterprise security posture.

## 1. Vulnerability Findings

| ID | Severity | Finding | Detailed Description |
| :--- | :--- | :--- | :--- |
| **VULN-CKGE-001** | **High** | **Graph/Query Injection Risk** | Direct use of untrusted strings (code entity names) to construct Node IDs/queries in Neptune. This allows for query injection, violating **data integrity and CMMC/SOX compliance**. |
| **VULN-CKGE-002** | **Medium** | **Lack of Input/Path Validation** | The ingestion process, if receiving paths, could allow **Directory Traversal** to read sensitive files outside the intended scope. |
| **VULN-CKGE-003** | **Low** | **Dependency Chain Risk** | Future reliance on `tree-sitter` bindings and other libraries introduces **Software Composition Analysis (SCA)** risk (CVEs). |

## 2. Detailed Fix and Remediation Plan (Implemented in V6)

### Remediation for VULN-CKGE-001: Graph Injection

**Action:** **IMMEDIATE PATCH (InputSanitizer)**
1.  **Sanitization Utility:** Implemented `InputSanitizer.sanitize_for_graph_id()` to strip or escape all special characters (quotes, backslashes, Gremlin keywords) from code entity names.
2.  **Application:** Applied sanitation universally to all Node IDs and property values derived from ingested code.

### Remediation for VULN-CKGE-002: Path Validation

**Action:** **Architectural Control**
1.  The Fargate task is designed to **only** receive **S3 URIs** as input, streaming data directly via Boto3, eliminating the risk of local file path manipulation.

### Remediation for VULN-CKGE-003: Dependency Risk

**Action:** **CI/CD Security Gate**
1.  The `pipeline_config.yml` now mandates **Amazon ECR Enhanced Scanning** and dedicated **SCA scanning** (e.g., Dependency-Track) during the build stage to fail deployment if critical CVEs are found.

## 3. Human Developer Next Steps (The 20% Hand-off)

1.  **Semantic Embedding Audit:** Manually review the vectors generated by the **Embedding Agent** to ensure zero context poisoning from internal policy documents.
2.  **Zero-Trust Policy Review:** A dedicated security architect must audit the IAM roles to ensure strict adherence to **Least Privilege** across the Neptune, OpenSearch, and Fargate VPCs.
