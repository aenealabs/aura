9-Layer Deployment Architecture

Aura deploys via a modular 9-layer CloudFormation architecture. Each layer is independently deployable with explicit dependencies, enabling parallel builds and targeted updates.

Deployment Cascade (Total: ~120-180 min dev, 180-240 min prod) Layer 0: Bootstrap (manual, one-time) ↓ Layer 1: Foundation (5-7 min) ├──→ Layer 2: Data (15-20 min) │ └──→ Layer 3: Compute (30-45 min) │ └──→ Layer 4: Application (10-15 min) │ └──→ Layer 6: Serverless (10-15 min) │ └──→ Layer 7: Sandbox (5-10 min) │ └──→ Layer 8: Security (5-10 min) │ └──→ Layer 9: Scanning (5-10 min) │ └──→ Layer 5: Observability (5-10 min, parallel with Data) └──→ Network Services & K8s Deploy

Layer 0: Bootstrap & Account Setup

3 templates · Account bootstrap, KMS, SNS, CodeBuild projects (20 total). One-time manual trigger per environment.

Layer 1: Foundation — VPC, IAM, Security, DNS

12 templates · VPC (10.0.0.0/16), 2-3 AZs, security groups, IAM roles, KMS keys, 10 VPC endpoints, Route 53, dnsmasq ECS Fargate, ECR base images, S3 build cache.

Layer 2: Data — Neptune, OpenSearch, DynamoDB, S3, Redis

13 templates · Neptune graph DB (provisioned), OpenSearch vector search, 25+ DynamoDB tables, S3 buckets (artifacts, logs, reports), ElastiCache Redis, context provenance, supply chain tables, runtime security tables.

Layer 3: Compute — EKS, Node Groups, ECR

10 templates · EKS 1.34 cluster, 3 node groups (general t3.large SPOT, memory r6i.xlarge ON_DEMAND, GPU g5.xlarge SPOT), ALB Controller, Cluster Autoscaler IRSA, 7 ECR application repos.

Layer 4: Application — APIs, LLM, Authentication, Frontend

12 templates · Bedrock model endpoints + guardrails, Cognito user pool, OAuth/OIDC multi-IdP, CloudFront + S3 marketing site, CloudFront + S3 docs portal, A2A infrastructure, Palantir IAM.

Layer 5: Observability — Monitoring, Secrets, Cost, DR

12 templates · Secrets Manager, CloudWatch dashboards (14) + alarms (175+), AWS Budgets, cost anomaly detection, disaster recovery (AWS Backup), OpenTelemetry collector, DEV/QA cost schedulers, GPU monitoring, alignment alerts.

Layer 6: Serverless — Lambda, Step Functions, EventBridge, Queues

20 templates · 45+ Lambda functions, 4 Step Functions state machines, custom EventBridge bus (60+ rules), SQS queues (Standard + FIFO), threat intel scheduler, HITL scheduler, chat assistant WebSocket, incident response, GPU scheduler, constitutional AI audit.

Layer 7: Sandbox — Ephemeral Environments, HITL, Training

13 templates · ECS Fargate sandbox cluster (zero-egress), HITL approval workflow, self-play SWE-RL training pipeline, self-service test environments (ADR-039) with budgets, monitoring, namespace controller, Service Catalog.

Layer 8: Security — Config, GuardDuty, Runtime Security

13 templates · AWS Config (17 managed rules), GuardDuty, CloudFormation drift detection, AURA-ATT&CK red team (75 techniques), semantic guardrails, capability governance, IAM security alerting, runtime security (interceptor, discovery, baselines, correlation).

Layer 9: Scanning Engine — Vulnerability Discovery

9 templates · DynamoDB (jobs + findings), S3 (artifacts + reports), ECS Fargate zero-egress cluster, Step Functions 7-stage pipeline, 2 ECR repos, CloudWatch dashboard + alarms, cleanup Lambda, EventBridge custom bus + routing rules.

3-Tier DNS Architecture (dnsmasq)

Tier 1: EKS DaemonSet
Local caching DNS on every EKS node. Resolves *.aura.local service endpoints (neptune.aura.local:8182, opensearch.aura.local:9200). DNSSEC validation enabled.
Tier 2: ECS Fargate (VPC-Wide)
Centralized DNS for non-EKS services. NLB-fronted ECS tasks. Threat intelligence blocklist integration. Private ECR image with vulnerability scanning.
Tier 3: Ephemeral Sandbox
Per-sandbox DNS for isolated test environments. Automatic provisioning/teardown with sandbox lifecycle. Zero leakage between sandboxes.

CI/CD Pipeline

Modular layer-based CodeBuild cascade with 20 projects, 38 buildspecs, and a 3-gate validation system. CodeBuild is the single source of truth — manual deployments are prohibited.

Critical Rules: (1) CodeBuild is the ONLY authorized deployment method. (2) Never trigger CodeBuild while another build is running. (3) No manual CloudFormation deployments. (4) Max 600 lines per buildspec.

Supply Chain Security (Mandatory)
All CI/CD builds use private ECR base images from aura-base-images/. Public images (Docker Hub, public ECR) are prohibited in production. Bootstrap-once pattern: pull from ECR Public Gallery, push to private ECR with vulnerability scanning.
Podman-First (ADR-049)
Podman is the primary container runtime for local development (avoids Docker Desktop licensing). Docker available as CI/CD fallback in CodeBuild. Always use --platform linux/amd64 on Apple Silicon.

Security Infrastructure

AWS Config compliance rules, GuardDuty threat detection, drift detection, WAF, KMS encryption, and runtime security monitoring.

AWS Config (17 Managed Rules)
CMMC/NIST/DoD compliance monitoring. Custom rules for EKS RBAC, pod security, and network policies. Auto-remediation for non-compliant resources. Real-time compliance dashboard.
GuardDuty
Network + API call analysis. EKS Audit Log integration. Security Hub findings aggregation. Automated SNS → Lambda → remediation response chain.
Drift Detection (Layer 8.3)
CloudFormation drift detection Lambda every 6 hours. CloudWatch dashboard + SNS alerts. Auto-rollback or HITL manual approval for remediation.
KMS Encryption
15+ customer-managed keys with automatic rotation. All databases, secrets, S3 buckets, and EBS volumes encrypted. Key policies scoped via kms:ViaService conditions.
WAFv2
Web Application Firewall rules for ALB + API Gateway. Rate limiting, SQL injection protection, XSS filtering, and geo-blocking capabilities.
Runtime Security (ADR-083)
Traffic interception, shadow agent detection, behavioral baselines, AURA-ATT&CK red team (75 techniques), runtime-to-code correlation via GraphRAG. 4 dedicated CloudFormation templates.

Cost Management

Multi-layer cost optimization including DEV cost scheduler, QA kill-switch, SPOT instances, and AWS Budget alarms.

DEV Cost Scheduler (Layer 5.11)
Lambda scales EKS nodegroups to 0 nightly at 1:00 UTC. Restores Mon-Fri at 12:00 UTC. Weekends fully off. General, memory, and GPU node groups managed.

Savings: ~$115-145/month (27-37% of compute)
SPOT Instances
All general + GPU EKS nodes use SPOT pricing. ~70% cost savings vs ON_DEMAND. Interruption rate ~1-2% (acceptable for dev). Memory nodes ON_DEMAND (ML workload reliability).
