Coverage & Configuration

Pytest configuration, coverage thresholds, and marker definitions that govern test execution.

70% Coverage Minimum — Non-Negotiable
The fail_under = 70 setting in pyproject.toml is a hard requirement. If coverage drops below 70%, tests must be added to increase coverage — the threshold must never be lowered. Violations require explicit written approval from the project owner.

Pytest Configuration (pyproject.toml)

# pytest settings [tool.pytest.ini_options] testpaths = ["tests"] python_files = ["test_*.py"] python_classes = ["Test*"] python_functions = ["test_*"] addopts = [ "-v", "--strict-markers", "--tb=short", "--cov=src", # Coverage source directory "--cov-report=term-missing", "--cov-report=html", ] [tool.coverage.report] fail_under = 70 # HARD MINIMUM — DO NOT LOWER show_missing = true precision = 2

Test Isolation Patterns

Rate Limiter Isolation
Autouse fixture (reset_rate_limiter_fixture) prevents state bleeding between tests. API rate limiter reset before/after each test execution.
Configuration State Isolation
Uses copy.deepcopy() for config returns. Prevents consumers from mutating shared configuration objects during test execution.
Module Isolation
@pytest.mark.forked for process-isolated tests. isolated_modules() context manager for temporary sys.modules mocking with automatic restoration.
macOS / Torch Safety
Detects PyTorch imports, forces spawn mode for multiprocessing on macOS (Objective-C safety). Schedules torch-dependent tests separately to prevent fork-related crashes.

Quick Test Commands

Common commands for running tests locally and in CI/CD environments.

Local Development

# Run all tests pytest # Unit tests only (fast feedback) pytest -m "not integration" # Integration tests only pytest -m integration # With HTML coverage report pytest --cov=src --cov-report=html # Parallel execution (auto-detect workers) pytest -n auto # Last failed tests first (fast feedback loop) pytest --lf --ff # E2E tests (requires real AWS services) RUN_E2E_TESTS=1 pytest -m e2e --no-cov # Specific test file pytest tests/test_agent_orchestrator.py -v

Load Testing (k6)

# Smoke test (default) ./tests/load/run-load-tests.sh # Stress test on API endpoints ./tests/load/run-load-tests.sh --profile stress --scenario api # Soak test (40 min memory leak detection) ./tests/load/run-load-tests.sh --profile soak --scenario database # Against staging environment ./tests/load/run-load-tests.sh --url https://api.aura.staging

Infrastructure Validation

# Validate all CloudFormation templates ./scripts/cfn-lint-wrapper.sh deploy/cloudformation/*.yaml # Validate IAM actions (check for typos/deprecated) python scripts/validate_iam_actions.py --report
