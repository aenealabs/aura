1. Architecture Overview

The system is secured by layered AI governance including Constitutional AI principles, capability governance, semantic guardrails, and configurable human-in-the-loop workflows.

Workflow Execution Phases

The System 2 Orchestrator executes a 10-phase workflow with checkpoint/resume support:

  1. INIT — Initialize task context and parameters
  2. PLANNING — Decompose task into sub-tasks via DAG
  3. MEMORY_LOAD — Load relevant cognitive memory
  4. CONTEXT_RETRIEVAL — Fetch code context via GraphRAG
  5. CODE_GENERATION — Coder Agent generates patches
  6. SECURITY_REVIEW — Security analysis of generated code
  7. VALIDATION — Validator Agent runs tests in sandbox
  8. REMEDIATION — Apply fixes based on review feedback
  9. MEMORY_STORE — Persist learnings to cognitive memory
  10. COMPLETED — Final result aggregation

Task Status Lifecycle

PENDING → IN_PROGRESS → COMPLETED
                    ↓            ↓
                  BLOCKED    AWAITING_HITL
                    ↓            ↓
                  FAILED     APPROVED/DENIED

Communication Flow

Agent A → A2A Gateway → A2A Security (validate) → A2A Registry (route) → Agent B ↓ ↓ SQS Queue (async tasks) SQS Queue (results) ↓ ↓ Agent Worker (dequeue & execute) Result Aggregation

Agent Runtime Service

The Agent Runtime Service (1,020 LOC) manages the full agent lifecycle: spawning, health monitoring, resource tracking, error handling, and graceful shutdown. Agents operate as async workers with configurable timeouts and automatic retry on transient failures.

5. Cognitive Memory Architecture

Aura implements a 5-type cognitive memory system combining Titan Neural Memory (ADR-024) with the Evo-Memory Framework (ADR-080) for persistent learning across agent interactions.

🧠
Working Memory

Active task context held during execution. Manages attention, token budgets, and intermediate results. Cleared on task completion.

📖
Episodic Memory

Full context snapshots of past interactions with TTL-based expiration. Records task parameters, results, and execution traces.

📚
Semantic Memory

Guardrails, patterns, and schemas learned from experience. Stores domain knowledge, coding conventions, and security rules.

⚙️
Procedural Memory

Action sequences with success tracking. Records tool invocation patterns, workflow recipes, and remediation strategies.

🔮
Neural Memory (Titan)

Deep MLP with test-time training (TTT) and MIRAS configuration. Provides sub-symbolic learning and pattern generalization.

10. Sandbox Architecture & Patch Workflow

The HITL Sandbox Architecture defines a 7-stage workflow for safe patch validation in isolated environments with human approval gates before production deployment.

Stage 1: Detection & Generation Reviewer Agent detects vulnerability → Coder Agent generates patch ↓ Stage 2: Sandbox Provisioning ECS/Fargate isolated environment → Deploy patched code → Configure security groups Environment ID: sandbox-{timestamp}-{uuid} ↓ Stage 3: Sandbox Testing Unit tests → Integration tests → Security tests (SAST/SCA) → Performance tests ↓ Stage 4: HITL Notification Create approval request → SNS notification → SES email with diff + results 24-hour timeout (configurable) ↓ Stage 5: Human Approval Decision Dashboard review → APPROVE / REJECT / REQUEST_CHANGES ↓ Stage 6: Production Deployment (Gated) Feature branch → CI/CD gates (SAST/SCA/ECR) → Staging → Production Monitor Agent tracks metrics for 24 hours ↓ Stage 7: Post-Deployment Verification Verify patch is live → Track errors/performance/security → Completion notice

    → DeployToProduction | HandleRejection | HandleTimeout
    → TeardownSandbox

15. Anomaly Detection & Behavioral Baselines

The ML-Based Anomaly Detection system (ADR-072) identifies 5 categories of anomalous agent behavior using statistical analysis, sequence n-gram detection, and honeypot capabilities with zero false positives.

5 Detection Categories

Volume Anomalies

Agent invoking 10x normal DANGEROUS tool calls. Spike in CRITICAL capability requests. Unusual burst patterns (100 calls / minute).

Sequence Anomalies

Tool invocation order deviating from learned workflows. Skipping prerequisite tools. Unusual tool combinations not seen in training.

Temporal Anomalies

Agent activity outside normal hours. Sudden capability usage after long dormancy. Timing patterns matching known attack signatures.

Context Anomalies

Production agent using sandbox-typical patterns. Test agent accessing production-like resources. Environment confusion indicators.

Cross-Agent Anomalies

Multiple agents simultaneously accessing same sensitive resource. Coordinated capability escalation attempts.

Response Actions

  • Z-score statistical detection for volume and temporal anomalies
  • Sequence n-gram analysis for workflow deviation detection
  • Honeypot capabilities with zero false positive rate
  • Automatic quarantine mechanism for confirmed threats
  • Natural language explanation (Bedrock-powered) for HITL review
  • HITL release workflow for quarantined operations
113 tests | ~1,400 LOC | 4 detector components

19. Agent Optimization & Enhancement

The Agent Optimization Roadmap (ADR-029) delivers performance enhancements that are enabled by default for all agents. Three phases are currently active in production.

Active Optimizations

Phase 1.3: Semantic Caching ENABLED

Caches responses for semantically identical prompts. Reduces LLM API calls by 30-40% and lowers latency for repeated queries.

Phase 2.2: Self-Reflection ENABLED

20. Recursive Context & Embedding Prediction

ADR-051 combines two December 2025 research paradigms — MIT CSAIL's Recursive Language Models for 100x context scaling and Meta FAIR's VL-JEPA for 2.85x inference efficiency — to dramatically expand agent capabilities.

Recursive Language Models (MIT CSAIL)

LLMs generate Python code to examine and decompose context, treating long prompts as external variables in a REPL. Handles 10M+ token inputs with cost scaling by task complexity, not input size.

# LLM generates analysis code:
def analyze_codebase(context):
    jwt_files = search_by_pattern(
        context, "jwt|authentication"
    )
    analysis = [
        analyze(section)
        for section in jwt_files
    ]
    return summarize(analysis)
100x context scaling | 2x improvement on long-context benchmarks

VL-JEPA (Meta FAIR)

Predicts embeddings instead of generating tokens. For non-generative tasks (classification, retrieval, routing), the Y-Decoder is skipped entirely, providing 2.85x efficiency.
